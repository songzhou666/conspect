# 质量审核体系

## 审核触发条件

质量审核在每个阶段的产物生成后自动触发。以下是各阶段的触发时机：

| 阶段 | 产物 | 触发时机 | 默认审核等级 |
|------|------|----------|-------------|
| 分析 | `_cs-analysis.md` 分析报告 | 文件写入完成后 | L1 |
| 设计 | `_cs-design.md` 设计方案 | 文件写入完成后 | L2 |
| 实现 | 源码文件 + 运行结果 | 文件写入 + 服务启动后 | L3 |
| 验证 | `_cs-verify.md` 验证报告 | 文件写入完成后 | L2 |

## 审核等级定义

| 等级 | 覆盖范围 | 适用阶段 | 说明 |
|------|----------|----------|------|
| L1 | 仅阻断检查 | 分析 | 检查产物是否存在、格式是否完整，不评分 |
| L2 | 阻断 + 关键维度评分 | 设计、验证 | 检查产物完整性 + 关键质量维度评分 |
| L3 | 全维度评分 | 实现 | 检查产物完整性 + 所有质量维度评分 |

## 等级判定标准

| 等级 | 评分范围 | 含义 | 处理方式 |
|------|----------|------|----------|
| A | ≥ 90 | 优秀 | 自动通过，进入下一阶段 |
| B | 75 - 89 | 良好 | 自动通过，记录改进建议 |
| C | 60 - 74 | 及格 | 需修复低分项后重审 |
| D | < 60 | 不及格 | 打回重新实现，重试计数 +1 |

## 阻断规则

1. **C 级以下打回修复**：评分 C 级或 D 级的阶段产物必须修复后重新提交审核
2. **P0 阻断（emoji 一票否决）**：任一阶段产物中发现 emoji，直接阻断，不参与评分，打回重做
3. **P0 阻断（关键文件缺失）**：关键产物文件缺失，直接阻断
4. **重试上限**：同一阶段最多重试 2 次，超过后标记为 `FAILED`
5. **阻断通知**：阻断时输出 `_cs-qa-blocked.md`，包含阻断原因和修复指引

```
┌──────────┐    通过(A/B)    ┌──────────┐
│  提交审核  │ ───────────→ │  进入下一阶段  │
│          │                │              │
│          │    不通过(C/D)  │              │
│          │ ───────────→ │  打回修复     │
└──────────┘                └──────┬───────┘
                                   │
                           ┌───────▼───────┐
                           │  重试次数 < 2  │────→ 修复后重新提交
                           │               │
                           │  重试次数 ≥ 2  │────→ 标记 FAILED
                           └───────────────┘
```

## 重审机制

当修复后的产物重新提交审核时：

1. **对比审核**：检查上次审核中记录的 P0（阻止性）问题是否已修复
2. **P0 未修复**：直接判定 D 级，不再进行其他维度评分
3. **P0 已修复**：按对应等级的全维度表重新评分
4. **回归检查**：确认修复未引入新的问题

## 审核报告格式

审核报告存储为 `_cs-qa-{phase}.md`，格式如下：

```markdown
# conspect 质量审核报告

## 基本信息
- **阶段**: 设计
- **审核等级**: L2
- **审核时间**: 2026-07-17 10:00:00
- **审核结果**: B（良好）

## 各维度评分

| 维度 | 权重 | 得分 | 加权得分 | 评价 |
|------|------|------|----------|------|
| 完整性 | 25% | 90 | 22.5 | 方案文档完整，包含所有要素 |
| 数据设计 | 25% | 80 | 20.0 | 维度识别基本正确，环比计算需确认 |
| 技术方案 | 25% | 85 | 21.25 | 技术路线可行，建议补充备选方案 |
| 一致性 | 25% | 85 | 21.25 | 与需求基本对齐 |

## 总分
**总得分：85.1 → B（良好）**

## 问题清单

| 编号 | 严重等级 | 问题描述 | 修复建议 |
|------|----------|----------|----------|
| P1 | P1（次要） | 环比计算基准周期未明确 | 在方案中补充环比计算的基准说明 |

## 审核结论
- [x] 自动通过
- [ ] 需修复后重审
- [ ] 阻断

## 审核人
AI Quality Auditor（自动）
```
