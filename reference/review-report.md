### P0-01：接力棒格式定义冲突（JSON vs Markdown）

| 字段 | 内容 |
|------|------|
| **严重等级** | P0 (阻断级) |
| **涉及文件** | `agents/00-master-controller.md` 第19-35行；`protocols/baton-protocol.md` 第20-49行 |
| **具体位置** | 00-master-controller.md 的"接力棒结构定义"展示 JSON 格式；baton-protocol.md 展示 Markdown 格式 |
| **问题描述** | 主控 Agent 定义的接力棒结构为 JSON 对象（`{"skill":"conspect","version":"1.0.0","status":"开始|分析|...",...}`），但接力棒协议文档定义的是 Markdown 格式（含 `## 元数据`、`- state: {开始/分析/...}`、`## 状态追踪` 等）。AI 在执行时无法确定使用哪种格式，两种格式的结构字段名也不同（JSON 用 `status`，Markdown 用 `state`；JSON 的 `current_agent` 在 Markdown 中不存在）。这是一个**致命的格式冲突**，直接破坏状态机的一致性。 |
| **建议修复方案** | 统一使用同一种格式。建议统一为 Markdown 格式（baton-protocol.md 中的版本），因为它更易读、可直接作为注释展示。删除 00-master-controller.md 中的 JSON 结构定义，改为引用 baton-protocol.md。如确需 JSON 格式，则需在所有引用处同步更新。 |

### P0-03：Master-Controller 状态路由表逻辑错误

| 字段 | 内容 |
|------|------|
| **严重等级** | P0 (阻断级) |
| **涉及文件** | `agents/00-master-controller.md` 第46-54行 |
| **具体位置** | 状态路由表："分析"行的"触发条件"为"用户确认分析结果"；"确认"行的"调用 Agent"为"02-designer-agent" |
| **问题描述** | 路由表中"分析"状态的触发条件写为"用户确认分析结果"，但"分析"状态之后应该是"确认"状态（等待用户确认），而路由表将用户确认作为"分析"的触发条件。实际上逻辑应该是：分析 Agent 完成 → QA 审核 → 自动进入"确认"状态 → 等待用户确认。同时，路由表中"确认"状态调用"02-designer-agent"也是错误的：确认是用户交互节点，不调用子 Agent，用户确认后才进入"设计"阶段调用设计 Agent。这张路由表的状态/触发条件/Agent 调用存在多处逻辑混淆。 |
| **建议修复方案** | 重写状态路由表，确保：分析完成后自动进入"确认"（等待用户）；用户确认后进入"设计"（调用 designer-agent）；设计完成且 QA 通过后进入"实现"；依此类推。触发条件严格区分"自动完成"和"用户确认"。 |

| 字段 | 内容 |
|------|------|
| **严重等级** | P0 (阻断级) |

### P1-01：QA 审核失败回退路径不一致

| 字段 | 内容 |

| **涉及文件** | `SKILL.md` 第481-485行；`agents/00-master-controller.md` 第197-229行伪代码；`protocols/phase-protocol.md` 第15-18行 |
| **具体位置** | 状态机回退路径 vs 伪代码中的回退实现 |
| **问题描述** | 三份文档定义了不同的回退路径：(1) SKILL.md 状态机：分析→开始, 确认→分析, 实现→设计, 验证→实现；(2) master-controller 伪代码：分析QA不通过→回退"开始", 设计QA不通过→回退"确认", 实现QA不通过→回退"设计", 验证QA不通过→回退"实现"；(3) phase-protocol 状态图：判定→设计(架构问题), 判定→实现(实现问题)。其中设计QA不通过回退到"确认"尤为不合理——"确认"是用户交互节点，回退到确认意味着让用户重新确认，而不是修复设计问题。正确的做法应是回退到"设计"阶段重做。 |
| **建议修复方案** | 统一所有回退路径：分析QA不通过→回退"分析"（非"开始"，保留上下文）；设计QA不通过→回退"设计"（非"确认"）；实现QA不通过→回退"实现"；验证不通过→根据错误类型路由到设计或实现。确保所有文档一致。 |
