# 渲染引擎详解

## 可视化引擎

conspect 使用 **ECharts 5.x** 作为可视化渲染引擎。

| 特性 | 说明 |
|------|------|
| 图表库 | Apache ECharts 5.x |
| 渲染方式 | Canvas（默认）/ SVG |
| 动画 | 入场动画默认开启，支持自定义缓动函数 |
| 响应式 | 容器尺寸变化时自动 resize |
| 主题 | 浅色商务主题，低饱和配色方案 |

## 图表类型映射规则表

以下规则指导 AI 根据数据特征自动选型图表：

| 数据特征 | 推荐图表 | 说明 |
|----------|----------|------|
| 1 个分类维度 + 1 个数值指标 | 柱状图 | 对比各分类的数值大小 |
| 1 个时间维度 + 1 个数值指标 | 折线图 | 展示时间趋势 |
| 1 个时间维度 + 多个数值指标 | 多系列折线图 | 多指标趋势对比 |
| 1 个分类维度 + 1 个占比指标 | 饼图 | 展示构成比例（分类 ≤ 7 类） |
| 1 个分类维度 + 1 个占比指标（多层级） | 旭日图 | 展示层级构成比例 |
| 2 个分类维度 + 1 个数值指标 | 分组柱状图 | 双维度交叉对比 |
| 2 个数值指标 | 散点图 | 展示两指标相关性 |
| 1 个分类维度 + 2 个数值指标 | 双轴图（柱+线） | 展示量级和趋势的复合关系 |
| 1 个结构维度 + 1 个数值指标 | 树图/矩形树图 | 展示层级结构下的数值分布 |
| 1 个地理维度 + 1 个数值指标 | 地图 | 地理分布可视化 |
| 1 个时间维度 + 1 个分类维度 + 1 个数值指标 | 热力图 | 展示时间×分类的密集矩阵 |
| 多个 KPI 指标卡片 | 指标卡 | 关键指标突出展示 |
| 1 个排序维度 + 1 个数值指标（TopN） | 条形图（横向） | 排名展示，标签更易阅读 |
| 无维度，纯数值统计 | 仪表盘 | 目标达成率展示 |
| 1 个分类维度 + 1 个数值指标（含累计%） | 帕累托图 | 二八分析，柱状为单项值+折线为累计占比 |
| 多维度信息量比较 | 雷达图 | 信息熵多维度对比 |
| 多指标分布对比 | 箱线图 | 展示 Q1/Q2/Q3/上下须 |
| 不平等程度分析 | 洛伦兹曲线 | 基尼系数可视化，含 45°公平线 |
| 多维关联热度 | 热力图 | 互信息矩阵可视化 |
| 趋势预测 | 多折线+置信区间 | 原始值+MA+预测+置信带 |
| 贪心选择过程 | 瀑布图 | 展示每个步骤的增量变化 |

### 选型优先级

```
数据特征匹配度 > 图表可读性 > 视觉多样性 > 实现复杂度
```

同一看板内图表类型由数据特征驱动，不做人为数量限制。每个分析角度选择最契合的图表类型，保持视觉统一性。

## 三种输出形态的渲染方案

### 1. 交互式 Web 看板

**实现方案**：通过 Python 后端生成完整 HTML，内嵌 ECharts CDN 引用。

**技术栈**：
- 后端：Python（Flask/FastAPI）
- 前端：ECharts 5.x + 原生 JavaScript
- 数据：后端渲染时将聚合 JSON 直接注入 HTML

**交互能力**：

| 交互 | 实现方式 | 说明 |
|------|----------|------|
| 维度筛选 | ECharts dataset + 下拉选择器 | 切换维度后图表联动更新 |
| 图表联动 | ECharts connect API | 同一维度的多个图表联动高亮 |
| Hover悬浮 | ECharts tooltip | 显示当前点的详细数据 |
| 下钻 | 点击事件 + 局部刷新 | 点击分类展开子维度 |
| 数据导出 | 按钮触发下载 CSV | 导出当前视图的聚合数据 |
| 全屏 | 浏览器 Fullscreen API | 看板全屏展示 |

### 2. 轻量化离线 HTML

**实现方案**：单文件 HTML，内联所有 CSS/JS，不依赖外部 CDN。

**特点**：

| 特性 | 说明 |
|------|------|
| 文件体积 | 控制在 500KB 以内（含 ECharts 精简版） |
| 外部依赖 | 零外部依赖，可离线打开 |
| 数据 | 仅写入聚合后的 JSON 数据 |
| 样式 | 内联 CSS，无外部样式表 |
| 兼容性 | 支持 Chrome/Firefox/Edge 最新版 |

**生成流程**：

```
选择精简模板 → 注入聚合JSON → 内联ECharts → 写入单HTML文件
```

### 3. PDF/高清 PNG

**实现方案**：使用 Playwright 无头浏览器打开生成的 HTML，截图输出。

**流程**：

```
生成离线HTML → Playwright打开 → 等待渲染完成 → 截图 → 输出PDF/PNG
```

**截图表配置**：

| 参数 | 值 |
|------|-----|
| 浏览器 | Chromium（无头模式） |
| 视口 | 1920 × 1080（可配） |
| 等待策略 | 网络空闲 + 动画完成（wait_until="networkidle"） |
| 输出格式 | PDF（A4 横向）/ PNG（高清 2x） |
| 多页处理 | 超过一页时自动分页 |

## 标准布局

```
┌──────────────────────────────────────────────────┐
│  [标题] 看板标题（左对齐）                日期     │  <- 头部
├──────────────────────────────────────────────────┤
│  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐       │
│  │ KPI1 │  │ KPI2 │  │ KPI3 │  │ KPI4 │       │  ← KPI 区
│  │ 数值  │  │ 数值  │  │ 数值  │  │ 数值  │       │
│  └──────┘  └──────┘  └──────┘  └──────┘       │
├──────────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐             │
│  │   图表 A     │  │   图表 B     │             │  ← 基础分析
│  │  (主趋势图)   │  │ (构成分析)    │             │
│  │ [解读]解读文字  │  │ [解读]解读文字  │             │  <- 数据解读
│  └──────────────┘  └──────────────┘             │
│  ┌──────────────┐  ┌──────────────┐             │
│  │   图表 C     │  │   图表 D     │             │
│  │  (对比分析)   │  │ (明细表格)    │             │
│  │  [解读]解读文字  │  │  [解读]解读文字  │             │
│  └──────────────┘  └──────────────┘             │
├──────────────────────────────────────────────────┤
│  [高级] 深度洞察（高级分析区）                        │  <- 高级分析
├──────────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐             │
│  │  帕累托图    │  │  雷达图      │             │
│  │  [解读]解读文字  │  │  [解读]解读文字  │             │
│  └──────────────┘  └──────────────┘             │
│  ┌──────────────────────────────┐               │
│  │  趋势+预测（全宽）            │               │
│  │  [解读]解读文字              │               │
│  └──────────────────────────────┘               │
├──────────────────────────────────────────────────┤
│  [结论] 数据结论与建议（Markdown 渲染）              │  <- 底部
│  • 核心发现 1                                     │
│  • 核心发现 2                                     │
└──────────────────────────────────────────────────┘
```

### 布局规则

1. **KPI 区**：最上方，至少 3 个关键指标，数据驱动无上限，每行不超过 4 个
2. **基础图表区**：最重要的趋势/对比图放在左上角（视觉焦点）
3. **高级分析区**：标题「深度洞察」，帕累托图 + 雷达图并排，趋势预测全景
4. **辅助图表**：按逻辑关联性排列，关联图表相邻摆放
5. **数据解读区**：每个图表卡片下方强制包含，浅蓝背景 + CSS 图标（禁止 emoji）+ 1-3 句白话业务解读
6. **底部结论**：AI 自动生成的文字结论，不少于 5 条核心发现（含高级分析洞察）
7. **留白**：各区块间距 16-24px，保持呼吸感
